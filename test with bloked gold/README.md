# Test with Locked Gold Outputs

This repository supports the paper **"Unified Knowledge Graphs for Adaptive Semantic Refinement in Text-to-SQL"**, submitted to the ISWC 2026 Industry Track, and is developed in collaboration with the **IBM T.J. Watson Research Center**.

This folder contains two scripts used to separate gold-data preparation from the verification of results produced by the pipeline.

## `extract_gold_fields.py`

This script extracts gold fields from an evaluation JSON file and saves them into a lighter file, `gold_only.json` by default.

The purpose is to keep only the information needed for later evaluation, in particular:

- `question_id`
- `question`
- `gold_result`
- `gold_error`

This way, the gold result is computed only once and can be reused in later runs, speeding up tests and reducing the cost of repeated evaluations.

Example usage:

```bash
python extract_gold_fields.py --input base.json --output gold_only.json
```

## `test_sql.py`

This script tests the result produced by the pipeline.

It reads:

- the reference dataset with questions and gold queries,
- the `base.json` file with pipeline predictions,
- the `gold_only.json` file generated by the previous script.

It then compares the results and computes evaluation metrics, including:

- exact match between predicted SQL and gold SQL,
- textual similarity,
- execution match on the SQLite database,
- counts of errors, timeouts, and missing cases.

In short:

- `extract_gold_fields.py` prepares and saves gold results once;
- `test_sql.py` uses those saved results to quickly verify whether the pipeline output matches the gold output.

Example usage:

```bash
python test_sql.py
```

## Recommended Flow

1. Run `extract_gold_fields.py` to create `gold_only.json`.
2. Run `test_sql.py` to compare pipeline predictions against the saved gold outputs.

## Notes

- File paths can be adapted if the dataset or predictions are stored in a different folder.
- If `gold_only.json` already exists and is up to date, gold extraction does not need to be rerun for every test.
